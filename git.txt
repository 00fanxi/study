Git1 - 생활코딩

git의 3가지 목적 

1. version관리

2. backup

컴퓨터의 불확실성 : 언제 고장날지 모른다.

컴퓨터의 확실성 : 언젠가는 고장난다.


지역 저장소 : 내 컴퓨터

원격 저장소 : github


push : 지역 저장소에서 원격 저장소로 

pull : 원격 저장소에서 지역 저장소로 fetch




3. collaborate 협업

두 사람이 같은 파일을 수정하거나 같은 파일의 같은 행을 수정하면 어떻게 될까

바로 이런 상황에서 같은 파일을 자동으로 합쳐주고

같은 행을 수정했으면 나중에 업로드한 사람한테 알아서 처리하라고 교통정리를 해주는 것이

git이 해주는 협업의 환상적인 기능.

(남의 파일을 덮어쓰거나 하는 일이 절대로 일어나지 않게 해줌)





git은 구체적인 제품이라기 보다는 제품군이다.

1.github.com이라는 회사에서 만든 github desktop 프로그램


2.tortoiseGit 귀여미 프로그램 : 윈도우 탐색기에 기생해서 사용됨(윈도우에서만 사용됨).


3.Sourcetree : 아주 강력한 툴. 단점 : 복잡함. 장점 : 자유도.


4. Git : 오리지널 프로그램. cmd를 이용해서 git 프로그램을 제어하는 독특한 프로그램.
